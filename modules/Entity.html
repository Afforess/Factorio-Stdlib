has (entity, field_name)
Tests whether an entity has access to the field

Parameters:

  • entity to test field access
  • field_name that should be tested for

Returns:

    true if the entity has access to the field, false if the entity threw an exception accessing the field
get_data (entity)
Gets user data from the entity, stored in a mod's global data.

The data will persist between loads, and will be removed for an entity when it becomes invalid

Parameters:

  • entity the entity to look up data for

Returns:

    the data, or nil if no data exists for the entity
set_data (entity, data)
Sets user data on the entity, stored in a mod's global data.

The data will persist between loads, and will be removed for an entity when it becomes invalid

Parameters:

  • entity the entity to set data for
  • data the data to set, or nil to delete the data associated with the entity

Returns:

    the previous data associated with the entity, or nil if the entity had no previous data
set_frozen (entity, mode)
Freezes an entity, by making it inactive, inoperable, and non-rotatable, or unfreezes by doing the reverse.

Parameters:

  • entity the entity to freeze or unfreeze
  • mode (optional) if true, freezes the entity, if false, unfreezes the entity. If not specified, is true.

Returns:

    entity passed into it
set_indestructible (entity, mode)
Makes an entity indestructible, so that it can not be damaged or mined by the player or enemy factions

Parameters:

  • entity the entity to set indestructible
  • mode (optional) if true, makes the entity indestructible, if false, makes the entity destructable. If not specified, is true.

Returns:

    entity passed into it
_are_equal (entity_a, entity_b)
Tests if two entities are equal

If they don't have reference equality and entitya has an 'equals' function, it will be called with entityb as the first argument

Parameters:

  • entity_a table
  • entity_b table

Returns:

    bool
